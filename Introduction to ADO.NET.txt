1. Что такое ADO.NET?
ADO.NET предоставляет собой технологию работы с данными, которая основана на платформе .NET Framework. ADO.NET - набор библиотек для взаимодействия с различными серверами БД различных поставщиков. ADO.NET позволяет отправлять запросы в базам данных, устанавливать подключение и тд.

2. Перечислите классы автономной модели ADO.NET
DataSet. Класс DataSet является ядром автономного режима доступа к данным в ADO.NET.
DataTable. Больше всего этот класс похож на таблицу БД. Он состоит из объектов DataColumn, DataRow, представляющих из себя строки и столбцы.
DataView. Это объект представлений базы данных.
DataRelation. Этот класс позволяет задавать отношения между различными таблицами, с помощью которых можно проверять соответствие данных из различных таблиц.

3. Основные классы в подключаемойчасти ADO.NET
Connection. Этот класс, позволяющий устанавливать подключение к источнику данных.
Transaction. Объект транзакций.
DataAdapter. Это своеобразный шлюз между автономными и подключенными аспектами ADO.NET. Он устанавливает подключение, и если подключение уже установлено, содержит достаточно информации, чтобы воспринимать данные автономных объектов и взаимодействовать с базой данных.
Command. Это класс представляющий исполняемую команду в базовом источнике данных.
Parameter. Объект параметр команды.
DataReader. Это эквивалент конвейерного курсора с возможностью только чтения данных в прямом направлении.

4. Что такое строка подключения?
Строка подключения это строка, которая содержит информацию, необходимую провайдеру для подключения к источнику данных.

5. Что собой представляет объект SqlConnection?
Объект SqlConnection представляет уникальный сеанс с источником данных SQL Server. В системе базы данных клиент / сервер это эквивалентно сетевому подключению к серверу.

6. Для чего предназначен объект SqlCommandStringBuilder?
StringBuilder позволяет создавать строку без перераспределения буфера для содержимого строки при каждой операции изменения
